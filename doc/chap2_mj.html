Here we include a list of all functions that are provided to the User.

2.1 Process Functions

2.1-1 DeclareProcess
‣ DeclareProcess( nrSteps[, parent, id][, title][, optrec] )( function )

Call this function to declare a process.

Returns the declared process. (see Chapter 1 for examples)

The argument nrSteps indicates the number of steps needed for the process to terminate. If just called with the argument nrSteps, the process will be declared as the root and be given an automatic id.

The optional argument parent must either be a process record, the id (string) of a valid process, or fail to declare the process as the root.

The optional argument id must be a unique string for the process.

The optional argument title must be a string or fail. If specified, the title will be printed in the progress bar.

The optional argument optrec must be a record with components that are valid display options. (see 2.2)

2.1-2 StartProcess
‣ StartProcess( process )( function )
‣ StartProcess( nrSteps[, parent, id][, title][, optrec] )( function )

Call this function before the start of a process. (For example exactly in the line above the for keyword in a loop)

Returns the started process (see Chapter 1 for examples)

If the process has already been declared, one can call this function with only one argument process, which must be either a process record or the id (string) of a valid process. Otherwise one has to use the function with arguments as specified in DeclareProcess (2.1-1) to declare the process.

2.1-3 UpdateProcess
‣ UpdateProcess( )( function )
‣ UpdateProcess( process[, title][, optrec] )( function )

Call this function at the end of a step of the given process. (For example exactly in the line above the od; keyword in a loop)

Returns the updated process (see Chapter 1 for examples)

The optional argument process must either be a process record or the id (string) of a valid process.

If not specified, process will be set to the root process. The optional argument title must be string or fail. If specified, the title of the process gets updated and will be printed in the prograss bar.

The optional argument optrec must be a record with components that are valid display options. (see Section 2.2)

2.2 Display Options

The methods DeclareProcess, StartProcess and UpdateProcess allow an optional argument optrec which must be a record and modifies the display output for the execution of only this single command.

For modifying the display output globally for all subsequent executions of the above functions see SetDisplayOptionsForProgressBar (2.2-2).

The following components of optrec are supported.

printTotalTime

true to print total time (of root process).

false to suppress total time. DEFAULT

printETA

true to print eta (expected time of arrival/completion). DEFAULT

false to suppress eta.

removeChildren

true to remove child process after its termination. DEFAULT

false to keep child process after its termination.

highlightCurStep

true to highlight current process.

false to suppress highlighting. DEFAULT

highlightColor

"red" for higlighting in red color. DEFAULT

"blue" for higlighting in blue color.

2.2-2 SetDisplayOptionsForProgressBar
‣ SetDisplayOptionsForProgressBar( optrec )( function )

sets the current global display options for progress bar.

The argument optrec must be a record with components that are valid display options. (see 2.2) The components for the current global display options are set to the values specified by the components in optrec.
